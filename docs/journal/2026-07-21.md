# ADL-8

## Objective

Transform the dashboard from static values into a live operational
health monitor.

## Completed

- Introduced IConfiguration into HealthStatusProvider.
- Read SQLite connection string from appsettings.json.
- Added SQLite connectivity check.
- Added structured exception logging.
- Added AlertCount property to HealthStatus.
- Executed first SQL query from the dashboard.
- Recovered accidental empty SQLite database.
- Identified and corrected working database location.
- Queried SQLite schema.
- Displayed live alert count on dashboard.

## Technical Concepts Learned

- IConfiguration
- Configuration Injection
- SqliteConnection
- ExecuteScalar()
- SQL COUNT(*)
- SQLite schema (sqlite_master)
- Structured logging
- Working directories
- Relative vs absolute paths
- SQLite database discovery

## Reflection

The dashboard now reports real operational data rather than
placeholder values.

The session also highlighted the importance of consistent database
locations and reinforced the need for repository separation as the
application grows.
