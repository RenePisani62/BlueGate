# ADL-7

## Objective

Begin converting the BlueGate web interface from a static demonstration into a service-driven architecture using ASP.NET Core Dependency Injection.

## Completed

- Moved HealthStatus into BlueGate.Common.
- Updated namespaces and project references.
- Introduced shared Models concept.
- Created IHealthStatusProvider interface.
- Implemented initial HealthStatusProvider.
- Registered provider with ASP.NET Core Dependency Injection container.
- Injected provider into IndexModel.
- Replaced direct HealthStatus construction with injected service.
- Successfully rendered data through Razor Pages.

## Technical Concepts Learned

- Shared projects
- Project references
- Namespaces
- Interfaces
- Dependency Injection
- Constructor Injection
- Razor Page Models
- Separation of Concerns

## Architectural Outcome

Previous implementation

Browser
→ Razor
→ PageModel
→ new HealthStatus()

Current implementation

Browser
→ Razor
→ PageModel
→ IHealthStatusProvider
→ HealthStatusProvider
→ HealthStatus

The current implementation remains backed by stub data but is now architecturally prepared for live health information.

## Reflection

This session represented a transition from procedural implementation toward service-oriented architecture. The PageModel no longer owns creation of application state and instead requests services from the ASP.NET Dependency Injection container.
