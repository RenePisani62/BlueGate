Shared Alert Model and Dashboard Integration

Status

Accepted

Context

BlueGate's dashboard required the ability to display the latest security alert.

The existing architecture had:

AlertRepository
HealthStatusProvider
HealthStatus

However:

AlertSummary existed within BlueGate.Server
HealthStatus existed within BlueGate.Common

This created an architectural dependency in the wrong direction.

Decision

Move AlertSummary into BlueGate.Common.Models.

Extend HealthStatus to include:

public AlertSummary? LatestAlert { get; init; }

Modify HealthStatusProvider to retrieve:

var latestAlert =
    _alertRepository.GetLatestAlert();

and populate:

LatestAlert = latestAlert

The dashboard now receives the latest alert as part of the health model.

Consequences

Benefits

Correct dependency direction maintained.
Shared models remain inside BlueGate.Common.
Dashboard now receives alert information without direct repository access.
Repository remains responsible for database access only.
Service remains responsible for orchestration.

Lessons Learned
Shared models belong in BlueGate.Common.
Repository Pattern continues to isolate SQL from presentation.
Service layer should orchestrate—not perform data access.
Product architecture should evolve before features are added.
