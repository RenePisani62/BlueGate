# ADR-8: Dashboard Health Checks and Live Alert Metrics

## Status

Accepted

## Context

The initial BlueGate dashboard displayed hard-coded values for
system health.

While useful for validating the Razor Pages architecture, the
dashboard did not reflect the true operational state of the
application.

BlueGate should display live health information derived from the
running environment.

## Decision

The dashboard now retrieves live health information through
HealthStatusProvider.

Implemented checks include:

- SQLite database connectivity
- Live alert count

The provider now:

- Reads the connection string from IConfiguration
- Opens a SQLite connection
- Executes SQL queries
- Returns a populated HealthStatus object

Exceptions are logged using ASP.NET Core logging.

## Consequences

Advantages

- Dashboard now reflects genuine application state.
- Introduces operational health monitoring.
- Establishes a reusable health-check pattern.
- Demonstrates configuration injection.
- Demonstrates structured logging.

Disadvantages

- HealthStatusProvider currently contains SQL logic.
- Future refactoring should move database operations into AlertRepository.

## Future Work

Create AlertRepository responsible for:

- GetAlertCount()
- GetLatestAlert()
- Save()
- Query operations

HealthStatusProvider will consume AlertRepository rather than
executing SQL directly.
