# ADR-011: Health Status Is the Authoritative Source for Dependency Availability

## Status

Accepted

## Date

18 July 2026

## Context

BlueGate displays operational health information for dependencies including
SQLite and the Sysmon Operational event log.

HealthMonitor was updated to perform genuine runtime checks through:

- AlertRepository.TestConnection()
- SysmonReader.IsAvailable()

During failure-path testing, the SQLite health check correctly returned false,
but the console continued to display the database as connected.

Investigation found that BlueGateAgent created a valid HealthStatus object but
then populated AgentCycleResult.DatabaseAvailable using a hard-coded true value.

This caused the verified result from HealthMonitor to be ignored.

## Decision

HealthMonitor will be the authoritative coordinator of dependency health checks.

HealthStatus will carry the results produced by HealthMonitor.

When BlueGateAgent creates AgentCycleResult, it must use values from HealthStatus
rather than hard-coded availability values.

The database mapping will therefore use:

```csharp
DatabaseAvailable = healthStatus.DatabaseConnected;
